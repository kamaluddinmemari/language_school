# راهنمای نصب و خلاصه تغییرات

## ۱. نصب و اجرا (بک‌اند)

```bash
cd language_school
pip install -r requirements.txt
python manage.py makemigrations
python manage.py migrate
python manage.py runserver
```

> چون وضعیت‌های جدیدی (`referred`, `confirmed`, `cancelled`) به مدل اضافه شده و `approved` حذف شده، اگه قبلاً توی `db.sqlite3` رکوردی با وضعیت `approved` ثبت کرده بودید، بهتره برای شروع تمیز (تا وقتی پروژه هنوز در حال توسعه‌ست) فایل `db.sqlite3` رو حذف کنید و دوباره `migrate` بزنید.

## ۲. ساخت اولین حساب مدیر (نکته‌ی مهم!)

چون الان ثبت‌نام عمومی (`/api/register/`) همیشه نقش دانش‌آموز می‌سازه (به خاطر فیکس امنیتی)، هیچ راه مستقیمی برای ساخت حساب مدیر از طریق API وجود ندارد. برای ساخت اولین مدیر:

```bash
python manage.py createsuperuser
```

بعد وارد `http://127.0.0.1:8000/admin/` بشید، با همون حساب لاگین کنید، روی رکورد خودتون در بخش Users بزنید و فیلد **role** رو از `دانش‌آموز` به `مدیر` تغییر بدید و ذخیره کنید. از این به بعد می‌تونید با همین حساب وارد پنل وب ادمین بشید و استادها رو از همونجا اضافه کنید.

## ۳. خلاصه تغییرات بک‌اند

- `core/settings.py`: حذف اپ ناموجود `teachers`، اضافه شدن `MEDIA_URL`/`MEDIA_ROOT`، تایم‌زون به `Asia/Tehran`
- `core/urls.py`: سرو کردن فایل‌های مدیا (عکس فیش، آواتار) در حالت DEBUG
- `accounts/models.py`: وضعیت‌های `referred`/`confirmed`/`cancelled` به `ClassRequest` اضافه شد، فیلد `accepted_teachers` (برای ردیابی تایید اولیه استادها)، `national_code` اختیاری شد
- `accounts/serializers.py`: رفع مشکل امنیتی (نقش دیگه از سمت کاربر قابل تعیین نیست)، اضافه شدن `TeacherSerializer` برای مدیریت استادها، نمایش `average_rating`
- `accounts/views.py` + `urls.py`: endpoint های کامل افزودن/ویرایش/حذف استاد (`/api/teachers/`, `/api/teachers/<id>/`)
- `classes/serializers.py`: بازنویسی کامل — سریالایزر جدا برای مدیر (کامل)، استاد (بدون اطلاعات تماس دانش‌آموز) و دانش‌آموز (نام استاد فقط بعد از تایید نهایی)
- `classes/views.py` + `urls.py`: فلوی دومرحله‌ای کامل:
  - `POST /api/classes/<id>/assign/` — ارجاع به یک یا چند استاد
  - `POST /api/classes/<id>/accept/` — تایید اولیه استاد
  - `POST /api/classes/<id>/decline/` — رد ارجاع توسط استاد
  - `POST /api/classes/<id>/finalize/` — تایید نهایی مدیر (با اطلاع‌رسانی به استادهای انتخاب‌نشده)
  - `POST /api/classes/<id>/cancel/` — کنسل کامل کلاس (اطلاع به دانش‌آموز + همه‌ی استادهای ارجاع‌شده)
  - `POST /api/classes/<id>/reject/`, `/complete/`, `/confirm/`, `/satisfaction/`, `/payment/` — مثل قبل، با تطبیق وضعیت‌های جدید

## ۴. خلاصه تغییرات پنل ادمین (React)

- `Dashboard.js`: تب‌های وضعیت به‌روزرسانی شد (ارجاع‌شده، تایید نهایی، کنسل‌شده)
- ستون استاد حالا نام واقعی استاد رو نشون می‌ده (قبلاً فقط ID خام بود)
- وقتی کلاس در وضعیت «ارجاع‌شده» باشه، لیست استادهایی که تایید اولیه کردن به صورت رادیو-باتن نشون داده می‌شه و دکمه‌ی «تایید نهایی» اضافه شد
- دکمه‌ی «کنسل کلاس» اضافه شد

## ۶. رابط کاربری اپ موبایل + قیمت‌گذاری (دور پنجم)

- بک‌اند: endpoint جدید `GET/PATCH /api/price-settings/` برای خوندن/تغییر نرخ پایه‌ی قیمت
- اپ موبایل: صفحه‌ی `home.tsx` کامل بازنویسی شد (اکشن‌های استاد و دانش‌آموز)، و دو صفحه‌ی جدید اضافه شد: `request-form.tsx` (ثبت درخواست کلاس) و `satisfaction.tsx` (امتیازدهی)

### ⚠️ قدم اجباری بعد از جایگزینی فایل‌های اپ موبایل
```bash
cd language-school-app
npx expo install expo-image-picker
```
بدون این دستور، صفحه‌ی ثبت درخواست کلاس (که برای انتخاب عکس فیش بهش نیاز دارد) اجرا نمی‌شود و اپ کرش می‌کند.

## ۷. موارد باز که هنوز نیاز به کار دارن (در اولویت بعدی)

- اپ موبایل (`language-school-app`) هنوز فرم ثبت درخواست دانش‌آموز و دکمه‌های قبول/رد/اتمام کلاس برای استاد رو ندارد
- گزارش آمار تجمیعی کلاس‌ها (بازه‌ی زمانی + هزینه‌ی کلی) هنوز پیاده نشده
- مدیریت `PriceSetting` (تغییر قیمت پایه) هنوز API ندارد
- فیلد `phone` روی مدل `User` هنوز `unique=True` نیست — پیشنهاد می‌شه برای جلوگیری از تکرار شماره موبایل این رو اضافه کنید (نیاز به migration دارد)
